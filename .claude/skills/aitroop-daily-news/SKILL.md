# AITroop Daily News

双源抓取 + Claude 原生分析，无需外部 API Key。

- **smol.ai**：AI 行业专项新闻聚合（高密度、高相关性）
- **90 个技术博客**：Karpathy 精选，覆盖工程/安全/观点等广度内容

输出格式参考 [ai-daily-skill](https://github.com/geekjourneyx/ai-daily-skill)：简洁卡片 + 核心摘要 + 标签体系。

---

## 工作流程

### Step 1：确认参数

如果用户未指定，使用默认值直接执行，**不必等待确认**：
- 时间范围：过去 **48h**
- 精选数量：**Top 15**
- 输出语言：**中文**
- 分类过滤：全部分类

用户也可指定：`"只看 AI 新闻"` / `"过去 24 小时"` / `"英文输出"` 等。

### Step 2：抓取新闻

**2A. 首先抓取 smol.ai**（AI 专项源，优先级最高）

```
https://news.smol.ai/rss.xml
```

用 WebFetch 抓取，提取时间窗口内的所有条目。smol.ai 每期是一个大型 newsletter 条目，包含当日 AI 行业全部重要新闻。

**2B. 并行抓取技术博客**（广度补充）

读取 `references/feeds.md` 获取 Feed 列表，每批并行抓取 10 个。只保留时间窗口内的文章，超时/404 静默跳过。

提取字段：`title`、`link`、`pubDate`、`description/content`

### Step 3：评分与分类

对每篇文章快速评估：

**评分（三维，各 1-10 分，总分 30）**

| 维度 | 高分标准 | 低分标准 |
|------|----------|----------|
| 相关性 | 技术从业者普遍关心的重大事件 | 与技术行业关联薄弱 |
| 质量 | 深度分析、原创洞见 | 浅尝辄止、纯转载 |
| 时效性 | 刚发布的重要工具/正在发生的事件 | 已过时或无时效价值 |

取总分 Top N 篇进入报告。

**分类**（每篇选一个，与 ai-daily-skill 对齐）：

| key | 中文名 | 图标 | 内容范围 |
|-----|--------|------|----------|
| model | 模型发布 | 🤖 | 新模型/版本更新/架构突破 |
| product | 产品动态 | 💼 | 新产品/功能/企业动态 |
| research | 研究前沿 | 📚 | 学术研究/技术突破/论文 |
| tools | 工具框架 | 🛠️ | 开发工具/开源项目/SDK |
| funding | 融资并购 | 💰 | 投资/收购/IPO |
| events | 行业事件 | 🏆 | 争议/政策/监管/观点 |

smol.ai 的内容优先归入 `model`/`product`/`research`；技术博客内容更多归入 `tools`/`events`。

### Step 4：生成摘要

对每篇入选文章生成：
- **中文标题**：自然意译，不超过 25 字
- **摘要正文**：2-4 句，直接切入核心，保留具体数据/技术名称/版本号
- **关键信息标签**：3-5 个中文或英文标签（公司名、技术名、主题词）

### Step 5：组装报告

按 `references/output-format.md` 的模板生成完整报告。

核心结构：
1. 标题 + 一句话摘要
2. 核心摘要（3-5 条要点）
3. 各分类文章卡片
4. 底部 hashtag 关键词

---

## 注意事项

- 同一分类文章超过 5 篇时，只展示得分最高的 5 篇
- smol.ai 的单篇 newsletter 可能包含多条独立新闻，拆分处理，各自独立成卡片
- 报告底部注明数据来源（smol.ai + 博客列表）和生成时间
- 如果用户要求保存文件，默认保存为 `./digest-YYYYMMDD.md`
